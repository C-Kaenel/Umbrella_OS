# Security Policy

## Supported Versions

Umbrella OS is currently in early development (Alpha). Only the latest commit on `main` is supported.

| Version | Supported |
|---|---|
| Main (Newest)| Yes |
| Older commits | No |

## Reporting a Vulnerability

Do not report security vulnerabilities via GitHub Issues.

If you discover a security vulnerability in Umbrella OS, please report it responsibly via the [Security Advisories](https://github.com/C-Kaenel/Umbrella_OS/security/advisories/new) page.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix if applicable

### What to expect

- Acknowledgement within 48 hours
- Status update within 7 days
- Credit in the changelog if desired

## Scope

The following are in scope:

- Boot process and init system
- Network configuration and MAC randomization
- Installer script
- Privacy protection mechanisms

## Philosophy

Umbrella OS is built with a privacy-first approach. Any vulnerability that could expose user identity, hardware identifiers, or network traffic is treated as critical.
